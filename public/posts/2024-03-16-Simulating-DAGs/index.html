Simulating Directed Acyclic Graphs (DAGs)

Directed Acyclic Graphs (DAGs)

DAGs are an incredibly succinct way to represent a set of condition

The basic premise

  • What is causal modeling

  • Why do they use DAG

    • The underlying structure of a DAG encodes the conditional dependencies of the variables present

  • In R

    • library(simDAG) can be used to generate data from DAGs

    • library(rCausalMGM) is a causal discovery package that attempts to determine the unknown DAG.

Say we are given the following model:

stateDiagram
    direction TB
    A --> C
    B --> C
    B --> D
    D --> E
    C --> E

Here nodes in the graph represent random variables and edges connecting nodes show the causal relationships between variables. Here for example, A and B have a causal effect on C, but D is only effected by B.

We can think of these relationships more rigorously as a Bayesian Network where the graph illustrates how to factorize the total joint probability (the probability that all these events occur simultaneously). Usually, the joint probability distribution represents all possible pairs of outcomes from the given random variables, but we can eliminate some combinations because the graph tells us they are conditionally independent. Writing out the joint probability distribution for this graph would give us:

P(A,B,C,D,E)=P(A)⋅P(B)⋅P(C|A,B)⋅P(D|B)⋅P(E|C,D)

Notice that each variable need only be conditioned on it's parents (nodes that point to another node). If we assume these variable take on specific probability distributions, we could generate data that corresponds to this DAG. Let's try it out!
